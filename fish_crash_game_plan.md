# 明太魚快手捕獲戰開發規劃

## 遊戲定位

新增一個獨立的滑動捕魚模式，暫定名稱為「明太魚快手捕獲戰」。玩法採用拋物線飛行與滑動捕捉概念，但名稱、畫面、角色、扣分物品及視覺效果均使用本專案自己的設計。

現有初級釣魚、高級釣魚與新模式應分開計分及排名，避免不同玩法的分數尺度互相影響。

## 核心玩法

- 每局 45 秒。
- 五種顏色的明太魚會從畫面底部，以不同角度、速度及高度飛入畫面。
- 玩家按住並滑動畫面，以手刀軌跡碰到魚即視為捕獲。
- 每條魚只能計分一次。
- 快速滑動時仍需進行連續軌跡碰撞判定，避免手指已劃過魚但沒有成功捕獲。
- 限制使用單一主要觸點，多點觸控不會重複累積分數。
- 魚掉回畫面外不扣分，但會中斷目前的連擊。
- 遊戲開始前顯示簡短滑動教學，正式開始後才出現角色與飛魚。

## 建議計分規則

| 行為 | 分數 |
| --- | ---: |
| 捕獲一般明太魚 | +1 |
| 捕獲與本局 Ping Doongs 同色的魚 | 額外 +3 |
| 同一刀捕獲 3 隻魚 | 額外 +3 |
| 同一刀捕獲 5 隻以上 | 額外 +8 |
| 同一刀剛好捕獲五種不同顏色 | 額外 +100，取代一般五隻連擊 +8 |
| 劃到扣分物品 | -10 |
| 最低分數 | 0 |

這個模式同時出現的魚量較大，因此不建議直接沿用現有同色魚 `+10` 的規則，否則分數容易快速膨脹。新模式使用獨立排行榜，不會影響初級與高級釣魚的計分方式。

同一刀同時碰到魚和扣分物品時，已捕獲的魚仍然得分，但另外執行扣分並將連擊歸零。

五色全捕獲的 `+100` 是連擊獎勵；五隻魚的基本分數與其中的幸運同色加分仍會照常計算。

## Ping Doongs 規則

- 每局由一隻 Ping Doongs 代表本局幸運色。
- 可沿用目前五隻 Ping Doongs 的固定公平輪替順序。
- 任何連續五局都會輪到全部五隻。
- 開始畫面不顯示 Ping Doongs，按下開始並完成倒數後才顯示。
- 畫面上只保留簡短的幸運色標示，避免狀態列過長。

## 扣分物品

建議使用符合活動風格、但容易和明太魚區分的「破掉的漁網」或「海洋垃圾」，不直接使用其他遊戲常見的炸彈造型。

- 使用深紫色或黑色外觀，外加紅色警告光圈。
- 外觀輪廓不能與任何一種明太魚相似。
- 劃到後扣 10 分、連擊歸零、短暫震動並讓畫面輕微晃動。
- 同一個扣分物品只能觸發一次。
- 遊戲前 5 秒不出現扣分物品，讓玩家先熟悉操作。
- 同一時間最多顯示一個扣分物品，避免手機畫面過於混亂。
- 生成位置不能完全遮住魚，必須保留可以閃避的滑動路線。

## 難度曲線

### 0–10 秒

- 每批出現 1–2 隻魚。
- 飛行速度較慢。
- 不出現扣分物品，或從第 5 秒後才開始出現。

### 10–25 秒

- 每批出現 2–4 隻魚。
- 開始混合不同飛行角度。
- 部分批次加入一個扣分物品。

### 25–35 秒

- 每批出現 3–5 隻魚。
- 提高飛行速度與角度差異。
- 增加同一刀捕獲多隻魚的機會。

### 最後 10 秒

- 進入「明太魚大爆發」。
- 每批出現 5–8 隻魚。
- 扣分物品仍限制同時最多一個。
- 使用明顯的倒數、音效與畫面提示。

## 操作與碰撞判定

遊戲區建議使用 Pointer Events：

- `pointerdown`：開始記錄手刀軌跡並鎖定主要觸點。
- `pointermove`：記錄連續座標並檢查上一點到目前點之間的線段。
- `pointerup`／`pointercancel`：結束本次手刀，計算單刀連擊。
- 非主要觸點直接忽略，防止多指同時滑動累積分數。
- 使用線段與圓形或橢圓形的碰撞判定，不能只檢查目前手指座標。
- 每個物件加入 `caught` 或 `triggered` 狀態，避免同一次或後續軌跡重複計分。

只有 Canvas 遊戲區設定 `touch-action: none`，頁面其他區域仍需保留正常捲動能力。

## 畫面與特效

- 明太魚沿拋物線飛行，並帶有輕微旋轉。
- 滑動時顯示短暫的發光手刀軌跡。
- 捕獲魚時播放星星、愛心或粉色水花粒子。
- 同色魚使用更明顯的粉色或金色閃光。
- 一刀多魚時在軌跡旁顯示連擊數與額外分數。
- 劃到扣分物品時使用紅色分數、短暫畫面晃動及不同音效。
- 支援 `prefers-reduced-motion`，降低容易造成不適的晃動與粒子效果。

## 音效與震動

- 沿用目前已完成的 iOS AudioContext 解鎖與播放型音訊 session。
- 開始按鈕的使用者手勢內完成音效解鎖。
- 捕獲一般魚、同色魚、連擊及扣分物品使用不同音效。
- 扣分物品使用較短且明顯的震動。
- 音效播放失敗不能阻止遊戲進行。

## 技術架構

建議建立獨立頁面與程式，避免增加現有 `FishGame` 的複雜度：

- `fish-swipe.html`
- `fish-swipe.js`
- `fish-swipe.css`

遊戲區使用 Canvas：

- 使用 `requestAnimationFrame` 更新拋物線、旋轉、粒子與滑動軌跡。
- 依照實際經過時間更新位置，避免不同裝置更新率造成速度差異。
- Canvas 像素比例最高限制為 `2x`，兼顧 Retina 清晰度與 iPhone 效能。
- 預先載入五張魚圖與扣分物品素材，完成後才允許開始遊戲。
- 頁面切到背景時暫停計時與動畫，回到前景後再恢復，避免玩家失去遊戲時間。
- 控制同時存在的飛行物與粒子數量，避免低階手機掉幀。

## 素材規劃

可以直接沿用：

- `assets/fish/rong_fish.webp`
- `assets/fish/bomi_fish.webp`
- `assets/fish/enji_fish.webp`
- `assets/fish/najoo_fish.webp`
- `assets/fish/hayoung_fish.webp`

建議新增：

- 一張透明背景的扣分物品素材，例如 `assets/fish/broken_net.webp`。
- 可選的閃光、水花或警告圖示；若使用 Canvas 繪製簡單粒子則不需要額外圖片。

## 遊戲流程

1. 玩家進入新模式頁面。
2. 顯示遊戲規則與滑動示範，尚不顯示 Ping Doongs。
3. 玩家按下開始，解鎖音效並進行倒數。
4. 顯示本局 Ping Doongs 與幸運色，開始 45 秒遊戲。
5. 時間結束後只顯示分數、捕獲數、最高連擊及帳號輸入。
6. 輸入帳號並結算後，進入新模式獨立排行榜與祝福畫面。
7. 送出祝福後直接回到重新挑戰畫面，不顯示額外的「繼續挑戰」按鈕。

## 排行榜與 Google Apps Script

新模式使用獨立遊戲代碼：

```text
fish_swipe
```

Google Apps Script 建議新增：

- 排行榜工作表：`FishSwipeScores`
- 祝福工作表：`FishSwipeCheers`
- 排行榜 action：`fish_swipe_leaderboard`
- 祝福 action：`fish_swipe_cheers`

需要同步修改：

- `google-apps-script.gs` 的 `GAME_SHEETS`、`GET_ACTIONS` 與 `normalizeGame()`。
- `leaderboard.js` 的 `gameConfigs`、網址參數白名單與四語系分頁名稱。
- `leaderboard.html` 新增「⚡ 快手明太魚」排行榜分頁。
- 本機 fallback 使用獨立的 `apink_fish_swipe_leaderboard` 與 `apink_fish_swipe_cheers`。

由於排行榜分數由瀏覽器送出，後端可以額外限制不合理的負分、非整數或超出合理上限的分數，但無法完全防止使用者修改前端程式。

## 多國語言

所有新增文字需一次完成繁體中文、英文、日文與韓文：

- 遊戲名稱與說明。
- 操作教學。
- 計分規則。
- 同色魚與連擊提示。
- 扣分提示。
- 倒數與時間結束文字。
- 結算、排行榜與祝福畫面。
- 排行榜分頁名稱。
- 圖片替代文字與 ARIA 標籤。

## 測試重點

- iPhone Safari 的滑動、音效、震動及頁面捲動。
- Android Chrome 的 Pointer Events 與多點觸控阻擋。
- 桌面滑鼠拖曳操作。
- 非常快速的滑動不能漏判魚。
- 同一條魚不能重複得分。
- 多指同時操作不能重複累積分數。
- 扣分物品不能因同一刀的多個座標重複扣分。
- 裝置切到背景後計時不應繼續流失。
- Canvas 在不同螢幕比例下不能裁掉 HUD 或生成區域。
- 長時間遊玩不應持續累積未清除的物件、音效節點與計時器。
- 中、英、日、韓文不能出現缺少翻譯、文字超出或排行榜分頁過長。

## 建議開發順序

1. 建立 Canvas 頁面、基本遊戲循環與響應式尺寸。
2. 完成單指滑動軌跡與連續碰撞判定。
3. 加入五色魚、拋物線、捕獲計分及連擊。
4. 加入扣分物品、難度曲線、粒子、音效與震動。
5. 完成開始、結算、帳號輸入、排行榜與祝福流程。
6. 更新 Google Apps Script 與排行榜獨立分頁。
7. 補齊四種語言。
8. 在 iPhone、Android 與桌面瀏覽器進行完整 QA。

## MVP 完成標準

- 五色魚能穩定飛入畫面並被單指滑動捕獲。
- 快速滑動不漏判，同一條魚不重複得分。
- 扣分物品能正確扣分且不重複觸發。
- 45 秒難度曲線與最後 10 秒爆發模式正常運作。
- iOS 音效與手機觸控正常。
- 結算、獨立排行榜、祝福留言及重新挑戰流程完整。
- 所有畫面具備繁體中文、英文、日文與韓文。
